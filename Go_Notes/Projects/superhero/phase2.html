🤔 What Is Docker?

Imagine PostgreSQL is software. Traditionally, installing it is painful. Docker solves this by running PostgreSQL in an isolated box called a Container.

1

Docker Terminology

This is extremely important — these concepts are the foundation of everything!

📄 Image

Think: Cake Recipe. A recipe is not cake. It only tells how to make cake.

Similarly, postgres:17 is an image. It is not running. It is only a blueprint.

📦 Container

Think: Actual Cake. Created from the recipe. Now it is real and edible!

Similarly, superhero_postgres is a running PostgreSQL instance — a container.

Real World Docker World Programming
Cake Recipe Image Blueprint / Template
Actual Cake Container Object / Instance

💡 Programming Analogy: Class → Image, Object → Container

⚙️ What Happens Internally?

When you run docker run postgres, Docker does these steps automatically:

1
Download Image

Fetches postgres:17 from Docker Hub if not already present.

2
Create Filesystem

Sets up an isolated file system for the container.

3
Create Isolated Network

Gives the container its own network interface.

4
Start PostgreSQL Process

Launches the PostgreSQL server inside the container.

5
Container Is Ready!

PostgreSQL is now running and accepting connections.

⚠️ The Container Problem

Containers are temporary by default. If you delete the container, all data is gone! 😱

❌ Without Volume

  • Container stores data internally
  • docker rm container
  • Container deleted
  • Data deleted
  • Everything gone! 💀

✅ With Volume

  • Container stores data in a Volume
  • docker rm container
  • Container deleted
  • Volume survives!
  • Data is safe! 🎉

📦 Volume Architecture 💾

📦
Container
⬇️
💾
Volume
Persistent storage on your machine
⬇️
🗄️
Database Files
Safe even if container dies!

💡 Think of it like this: A volume is a USB drive that lives on your machine. Even if you throw away the computer, the USB still has your files!

🔌 Port Mapping

Inside the container, PostgreSQL listens on port 5432. But your laptop cannot directly access container ports. So we expose it!

Port Mapping Explained

💻
Host Machine
localhost:5432
↔️
🔌
Port Mapping
5432 → 5432
↔️
📦
Container
Port 5432

5432:5432 means: "When I visit localhost:5432 on my machine, send that to port 5432 inside the container."

🐳 Docker Compose

Instead of typing long docker run ... commands every day, we write a docker-compose.yml file. Then: docker compose up — everything starts!

1

Create Project Folder

mkdir superhero-api

cd superhero-api

This creates our project home. Everything we build will live inside superhero-api/.

2

Create .env File

Why? Hardcoding values is bad! We store configuration separately.

Instead of password = "admin123" in our code, we put it in .env.

# .env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_DB=superhero_db
POSTGRES_PORT=5432
POSTGRES_USER

Database username: postgres

POSTGRES_PASSWORD

Password: postgres (for learning only!)

POSTGRES_DB

Database name: superhero_db

POSTGRES_PORT

Exposed port: 5432

3

Create docker-compose.yml

This file describes: which containers to start, how to configure them, how to network them, and which volumes to use.

services:
  postgres:
    image: postgres:17
    container_name: superhero_postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: superhero_db
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:

🔍 docker-compose.yml Breakdown 🔎

1
services:

Defines containers. Currently just postgres. Later we could add go app, redis, etc.

2
image: postgres:17

Tells Docker: "Use PostgreSQL version 17"

3
container_name: superhero_postgres

Friendly name instead of a random generated one.

4
environment:

Variables passed into PostgreSQL. PostgreSQL reads these when starting.

5
ports: "5432:5432"

Host 5432 → Container 5432. Your laptop can now connect!

6
volumes:

PostgreSQL stores files in /var/lib/postgresql/data. We connect that to postgres_data volume. Data survives container deletion!

4

Start PostgreSQL

docker compose up -d

What does this command do? The -d flag means "detached mode" — runs in the background so your terminal stays free.

Internally, Docker does this:

1️⃣Reads docker-compose.yml
2️⃣Checks if postgres:17 image exists
3️⃣Downloads image if needed
4️⃣Creates postgres_data volume
5️⃣Creates network
6️⃣Creates superhero_postgres container
7️⃣Starts PostgreSQL process — Done!

✅ Verify Container Is Running

docker ps

Expected output:

You should see superhero_postgres with status Up. That means PostgreSQL is alive and ready!

5

Enter the Container

docker exec -it superhero_postgres bash

docker exec means: "Run a command inside a running container". The -it flags give you an interactive terminal.

You are now inside Linux running inside the container! Your prompt will change to something like root@container:/#

6

Open PostgreSQL CLI

Inside the container, run:

psql -U postgres

🤔 What Is psql?

The official PostgreSQL terminal client. It lets you run SQL commands directly!

CREATE DATABASE CREATE TABLE SELECT INSERT UPDATE DELETE

📋 Show All Databases

\l

Expected output:

  • postgres (default)
  • template0 (system template)
  • template1 (system template)
  • superhero_db ← Our database!

🔗 Connect to Our Database

\c superhero_db

\c means "connect". Now you are inside superhero_db and ready to create tables!

🗄️ Database Design

We need a superheroes table. Let us create it with all the right columns and constraints!

Create the superheroes Table 📊

CREATE TABLE superheroes (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    real_name VARCHAR(100) NOT NULL,
    universe VARCHAR(50) NOT NULL,
    power_level INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

🔍 Deep Explanation of Each Column 🔎

1
id SERIAL

PostgreSQL auto-increments this. Insert Iron Man → ID becomes 1. Insert Batman → ID becomes 2. Automatically!

2
PRIMARY KEY

Guarantees uniqueness. You cannot have two rows with id = 1. Ever.

3
VARCHAR(100)

Variable length string. Stores text up to 100 characters. Example: "Iron Man"

4
NOT NULL

Required field. Cannot be empty. Every superhero MUST have a name!

5
INTEGER

Stores whole numbers. Example: 95, 88, 70

6
TIMESTAMP DEFAULT CURRENT_TIMESTAMP

Stores date and time. Automatically sets to the current time when a row is created. Example: 2026-06-03 16:30:00

✅ Verify Table Exists

\dt

Expected output:

You should see superheroes listed. That means your table was created successfully!

📋 See Full Schema

\d superheroes

PostgreSQL shows you: Columns, Types, Constraints, and Indexes — everything about your table!

🦸‍♂️ Insert Our First Superhero!

Let us add Iron Man to our database. This is the moment of truth! 🎉

INSERT Statement

INSERT INTO superheroes
(name, real_name, universe, power_level)
VALUES
('Iron Man', 'Tony Stark', 'Marvel', 95);

⚙️ What Happens Internally? 🔧

1️⃣PostgreSQL receives the SQL
2️⃣Validates syntax
3️⃣Finds the superheroes table
4️⃣Creates a new row
5️⃣Auto-generates id = 1
6️⃣Stores row on disk (via volume!)
7️⃣Returns success ✅
