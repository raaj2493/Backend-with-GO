What is a Backend — Physical + Logical

📡 Physical Reality

A computer (server) listening for requests (HTTP, WebSocket, gRPC) on an open port (80/443) accessible over the internet. It could be an AWS EC2 instance, a bare-metal machine, or a container.

📦 Role (Serving)

Provides static files (HTML, images) or dynamic JSON data. Also accepts data sent by clients. Not just code — it's a machine running that code, accessible via network.

💡 Key Insight: A backend is both the software (API logic) AND the hardware/virtual machine that hosts it. Without network reachability, it's just a local process.

❤️real-world use case

Why Do We Need Backends? The Instagram “Like” Story

You tap ❤️ on a friend's post → seconds later your friend gets a notification. What happens in between?

  • App sends request to server with post_id + user_id
  • Server identifies you, persists "User A liked Post B" in database
  • Server looks up post owner → friend's user ID
  • Server triggers notification to friend's device (push notification)
🗄️ Centralized role: DATA

Fetch data · Receive data · Persist data · Coordinate between users. Any action that deals with data belongs on the backend.

✨ Single word summary: The backend is the single source of truth for all users, posts, and relationships.
🧩frontend vs backend — the hard limits

Why Can’t We Just Do Everything on the Frontend?

Browsers are powerful, but they are intentionally sandboxed and restricted. Here’s why backend servers are irreplaceable.

🔒 1. Security & Sandboxing

Browsers are isolated from the OS, file system, and environment variables. Malicious JS can't steal local files. Backend has full system access intentionally.

🌐 2. CORS Restrictions

Same-Origin Policy prevents frontend from calling arbitrary APIs unless backend allows it with CORS headers. Backend servers have no such restriction.

🗄️ 3. Database Connectivity

Backends maintain persistent connection pools (e.g., PostgreSQL driver). Browsers cannot keep long-lived DB sockets — would overwhelm databases with millions of connections.

⚡ 4. Computing Power

Frontend devices vary wildly (low-end phones). Heavy business logic would lag or crash. Centralized backend can be scaled vertically (more RAM/CPU).

🎯 Conclusion: Even if technically possible, putting backend logic on the frontend is insecure, unscalable, and breaks database/resource boundaries.

🌐 DNS Records

A (IPv4), AAAA (IPv6), CNAME (alias). Each record directs traffic to infrastructure.

🔥 Security Groups

Stateful firewall at AWS level. Allow/deny rules for ports, IP ranges. First line of defense.

↪️ Reverse Proxy (Nginx)

SSL termination, load balancing, serving static assets, routing to internal services (Node, Python, Go).

📦 PM2 Process Manager

Keeps Node.js apps alive, restarts on crash, log management, cluster mode.

🔐 Port 80/443

HTTP & HTTPS default ports. Security groups must open these for public traffic.

🏗️ EC2 Virtual Machine

Cloud server with public IP, elastic network interfaces, accessible globally.
